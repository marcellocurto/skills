# Wild Frontend

Create a highly original, visually ambitious frontend interface that is allowed to ignore the existing product design system, component library, and visual conventions.

Use this skill only when explicitly invoked. This is for creative exploration, concept art as code, standout artifacts, experimental landing pages, posters, visual prototypes, and interfaces where memorability matters more than product consistency.

## Core Principle

Treat the interface as a designed artifact, not just a product surface.

The goal is to produce something visually memorable, conceptually clear, and technically functional. Push the design direction hard. Avoid safe defaults.

## Creative Direction

Before coding, choose a strong visual concept.

Define:

- **Concept** — What is the interface’s central idea?
- **Mood** — What should it feel like?
- **World** — What visual universe does it belong to?
- **Signature detail** — What is the one thing someone will remember?
- **Interaction style** — How should the interface move, respond, or reveal itself?

Commit to a direction such as:

- brutalist editorial
- cinematic sci-fi console
- playful toy interface
- dense cyberpunk dashboard
- luxury fashion microsite
- organic botanical system
- retro operating system
- maximalist poster layout
- generative art interface
- analog paper-and-ink workspace
- museum exhibit panel
- arcade machine UI
- surreal glass-and-light environment
- industrial control room
- weird experimental zine
- soft claymorphism
- neo-baroque data display
- monochrome Swiss poster grid
- dreamlike gradient world
- tactile scrapbook interface

Do not mix every idea together. Pick one concept and execute it with discipline.

## Design Moves

Use bold, intentional choices across the full interface.

### Typography

Choose typography with character.

Use type to create identity through:

- oversized display headlines
- unusual scale contrast
- expressive letter spacing
- editorial pull quotes
- compressed labels
- huge numbers
- vertical or rotated text
- mono/typewriter details
- delicate serif contrast
- brutal utility typography
- poster-like hierarchy

If external fonts are unavailable, create distinction through scale, weight, spacing, casing, and layout.

### Color

Build a committed palette.

Use:

- one dominant atmosphere
- one or two sharp accents
- dramatic contrast
- monochrome with a single signal color
- acidic highlights
- muted editorial tones
- deep cinematic darks
- warm analog neutrals
- strange but controlled gradients
- status colors only when meaningful

Color should create a world, not merely decorate components.

### Layout

Break predictable SaaS structure.

Use:

- asymmetry
- overlapping layers
- diagonal flow
- poster composition
- split screens
- dense information clusters
- oversized hero sections
- floating panels
- editorial columns
- irregular grids
- anchored navigation
- dramatic negative space
- layered foreground/background systems

The layout should guide attention while feeling custom.

### Surface and Texture

Make the screen feel physically or atmospherically specific.

Use:

- grain
- noise
- glows
- scanlines
- paper texture
- glass layers
- hard shadows
- soft depth
- geometric patterning
- radial light
- borders and frames
- masks and cutouts
- ornamental dividers
- illustrated empty states
- custom badges and labels

Effects should reinforce the chosen concept.

### Motion

Use motion as part of the design language.

Consider:

- staged entrance sequences
- hover distortions
- reveal masks
- sliding panels
- kinetic typography
- parallax depth
- animated gradients
- pulsing status elements
- cursor-follow effects
- springy interactions
- cinematic transitions

Prefer a few memorable motions over constant noise. Respect reduced-motion preferences when practical.

## Functional Standard

Even when visually wild, the interface should still work.

Ensure:

- clear primary action
- readable important text
- responsive behavior
- usable states
- accessible focus treatment
- reasonable contrast for core content
- keyboard-accessible controls when interactive
- no broken layout at common viewport sizes
- no decorative effect that prevents use

Wild does not mean careless.

## Implementation Style

Write real working code in the requested stack.

Use whatever is appropriate for the artifact:

- HTML/CSS/JS
- React
- Vue
- Tailwind
- CSS modules
- SVG
- Canvas
- CSS animations
- component-local styling
- generated visual elements

You may create one-off components, custom CSS, unusual layout systems, decorative layers, and bespoke interactions.

Do not feel constrained by existing product components unless the user specifically requests compatibility.

## Output Expectations

When producing code:

- implement the actual interface, not just a description
- include the visual system directly in the code
- make the result cohesive
- make the design direction obvious
- include responsive behavior
- include interaction states
- keep the code understandable enough to modify

When explaining the result, briefly name the chosen concept and the signature design moves.

## Avoid

- safe SaaS dashboard defaults
- generic hero sections
- timid palettes
- random decoration without a concept
- bland centered cards
- interchangeable purple gradients
- copying the existing codebase style
- apologizing for being bold
- producing a design that looks like every other generated UI

## Quality Bar

The result should feel like a specific art direction brought to life in frontend code.

It should be surprising, cohesive, functional, and memorable.
